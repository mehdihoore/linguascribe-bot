# ربات LinguaScribe 🎙️

**[English README 🇬🇧 →](README.md)**

ربات تلگرامی که پیام‌های صوتی طولانی، فایل‌های صوتی و ویدیویی را به متن پیاده‌سازی‌شده‌ی دقیق، ترجمه فارسی، زیرنویس SRT و خلاصه‌ای جامع و آماده برای مطالعه تبدیل می‌کند — با استفاده از مدل‌های Gemini گوگل.

## نکته‌ی اصلی این ربات: برای فایل های صوتی طولانی بهینه شده است (طولانی‌تر از 25 دقیقه)
بیشتر ربات‌های پیاده‌سازی متن، به محض این‌که فایلی بیشتر از چند دقیقه به آن‌ها بدهید از کار می‌افتند. این ربات دقیقاً برای مقابله با همین مشکل ساخته شده و از پس فایل‌های **۲، ۳ و حتی بیش از ۱۰ ساعت** بر می‌آید — یک جلسه کامل کلاس، یک قسمت کامل پادکست، یک جلسه چند ساعته، هر چیزی که بدهید.

راز کار این‌جاست: به جای این‌که کل فایل را یک‌جا به Gemini بدهد، ربات ابتدا مدت‌زمان واقعی فایل صوتی را اندازه می‌گیرد و اگر بیشتر از ۲۵ دقیقه بود، آن را به بخش‌های پیاپی حدوداً ۲۵ دقیقه‌ای تقسیم می‌کند، هر بخش را جداگانه پیاده‌سازی می‌کند، و در پایان همه را دوباره کنار هم می‌گذارد تا یک متن پیوسته، یک فایل SRT کامل و یک خلاصه‌ی یک‌پارچه تحویل شما دهد. یک سخنرانی ۶ ساعته می‌فرستید، یک متن تمیز و یک خلاصه پس می‌گیرید — تقسیم‌بندی پشت پرده و بدون این‌که متوجه شوید انجام می‌شود. یک فایل ۱۰ ساعته فقط یعنی تعداد بخش‌های بیشتر و کمی صبر بیشتر، نه یک شکست.

این ربات طوری طراحی شده که به‌صورت رایگان درون **Google Colab** اجرا شود؛ یعنی از خود Colab به‌عنوان میزبان ربات و از گوگل درایو به‌عنوان محل ذخیره‌سازی دائمی نتایج استفاده می‌کند.

## قابلیت‌ها

- ⏱️ **پردازش واقعی فایل‌های طولانی** — تقسیم‌بندی و بازچینی خودکار یعنی فایل‌های چند ساعته یک استثنا نیستند، بلکه یک کاربرد عادی این ربات هستند
- 🎤 **پیاده‌سازی متن (Transcription)** — تبدیل دقیق گفتار به متن، به زبان اصلی فایل صوتی، همراه با برچسب‌گذاری اولیه گوینده‌ها در صورت وجود چند گوینده قابل‌تشخیص
- 🌐 **ترجمه به فارسی** — ترجمه خودکار هر متن پیاده‌سازی‌شده به فارسی
- 🎬 **تولید زیرنویس SRT** — بخش‌بندی خودکار و زمان‌بندی‌شده‌ی متن فارسی برای تولید فایل زیرنویس
- 📝 **خلاصه‌سازی آماده برای آزمون** — خلاصه‌های ساختاریافته و بلند به زبان فارسی (شامل مفاهیم کلیدی، استدلال‌ها، مثال‌ها و تحلیل) که مناسب آمادگی برای آزمون‌های ورودی دانشگاه است
- 📹 **پشتیبانی از ویدیو** — استخراج و فشرده‌سازی خودکار صدا از فایل‌های ویدیویی پیش از پردازش
- 🗜️ **پشتیبانی از فایل ZIP** — استخراج فایل‌های ZIP و پردازش دسته‌ای هر فایل صوتی یا متنی موجود در آن
- 🔑 **چرخش خودکار کلید API** — استفاده چرخشی از حداکثر ۵ کلید API جمینای و چند مدل جایگزین (Gemini/Gemma) برای عبور از محدودیت نرخ درخواست و خطاهای موقتی
- 💾 **پشتیبان‌گیری در گوگل درایو** — هر متن پیاده‌سازی‌شده و خلاصه به‌صورت خودکار در پوشه‌ی `STTCOLAB` در گوگل درایو کاربر ذخیره می‌شود

## نحوه عملکرد

```
پیام تلگرام (صوت/فایل صوتی/ویدیو/ZIP)
        │
        ▼
 دانلود و یکسان‌سازی (pydub/FFmpeg → مونو، ۱۶ کیلوهرتز، ogg/opus)
        │
        ▼
 آیا طولانی‌تر از ۲۵ دقیقه است؟ ──▶ تقسیم به بخش‌های ~۲۵ دقیقه‌ای
        │                                       │
        ▼                                       ▼
 پیاده‌سازی متن با Gemini (به زبان اصلی، به‌ازای هر بخش در صورت تقسیم)
        │
        ├──▶ ترجمه به فارسی ──▶ بخش‌بندی ──▶ تولید فایل SRT
        │
        └──▶ خلاصه‌سازی جامع فارسی (ساختار آماده‌سازی برای آزمون)
        │
        ▼
 اتصال دوباره‌ی بخش‌ها به یک متن/SRT/خلاصه‌ی یک‌پارچه
        │
        ▼
 ارسال نتایج به تلگرام + ذخیره نسخه در گوگل درایو
```

تمام درخواست‌های Gemini از یک تابع مرکزی (`gemini_request_with_retry`) عبور می‌کنند که به‌صورت خودکار بین کلیدهای API و فهرستی از مدل‌های جایگزین برای هر وظیفه می‌چرخد؛ به همین دلیل محدودیت نرخ درخواست یا در دسترس نبودن موقت یک مدل، کل ربات را از کار نمی‌اندازد — این موضوع در فایل‌های طولانی اهمیت بیشتری هم پیدا می‌کند، چون افتادن یک بخش از بین بیست بخش نباید نوزده بخش دیگر را هم خراب کند.

## پیش‌نیازها

- یک حساب تلگرام و یک توکن ربات از [@BotFather](https://t.me/BotFather)
- **API ID و API hash** تلگرام از [my.telegram.org](https://my.telegram.org)
- یک یا چند **کلید API جمینای** از [Google AI Studio](https://aistudio.google.com/apikey)
- یک حساب گوگل (برای Colab و درایو، در صورت اجرا به شکل پیش‌فرض پیشنهادی)

## راه‌اندازی (Google Colab — روش پیشنهادی)

1. فایل `main.py` را به‌عنوان نوت‌بوک Colab باز کنید (یا محتوای آن را در یک سلول نوت‌بوک جدید کپی کنید).
2. در Colab، پنل 🔑 **Secrets** را (نوار کناری سمت چپ) باز کرده و موارد زیر را اضافه کنید:

   | نام Secret | مقدار |
   |---|---|
   | `TELEGRAM_API_ID` | شناسه عددی API شما |
   | `TELEGRAM_API_HASH` | هش API شما |
   | `TELEGRAM_BOT_TTS` | توکن ربات دریافتی از BotFather |
   | `GOOGLE_API_KEY_1` | یک کلید API جمینای |
   | `GOOGLE_API_KEY_2` تا `GOOGLE_API_KEY_5` | *(اختیاری)* کلیدهای اضافی — هرچه کلید بیشتری اضافه کنید، پردازش فایل‌های خیلی طولانی روان‌تر پیش می‌رود |

3. سلول را اجرا کنید. در اولین اجرا، Colab از شما اجازه دسترسی به گوگل درایو را می‌خواهد — آن را تأیید کنید تا نتایج به‌صورت خودکار پشتیبان‌گیری شوند.
4. به محض مشاهده‌ی پیام `Bot @yourbotname started successfully!` در لاگ‌ها، می‌توانید به ربات خود در تلگرام پیام دهید.

## راه‌اندازی (محلی / بدون Colab)

منطق اصلی ربات وابستگی سختی به Colab ندارد، جز نحوه‌ی بارگذاری Secret ها و اتصال به درایو. برای اجرا در محیط دیگر:

1. `pip install -r requirements.txt`
2. [FFmpeg](https://ffmpeg.org/download.html) را نصب کنید (مورد نیاز `pydub` برای پردازش صوت و ویدیو) و مطمئن شوید در `PATH` سیستم شما قرار دارد.
3. فراخوانی‌های `google.colab.userdata` و `google.colab.drive` را در ابتدای `main.py` با روش دلخواه خود برای بارگذاری Secret ها جایگزین کنید (مثلاً `python-dotenv` همراه با فایل `.env`) و مرحله‌ی اتصال به درایو را حذف کرده یا به یک پوشه محلی هدایت کنید.
4. اجرا کنید: `python main.py`

## نحوه استفاده

پس از اجرای ربات، در تلگرام به آن پیام دهید:

- `/start` — پیام خوش‌آمدگویی و معرفی قابلیت‌ها
- `/help` — راهنمای استفاده
- ارسال یک **پیام صوتی، فایل صوتی، ویدیو یا فایل ZIP** — حتی فایل‌های چند ساعته. اگر فایل طولانی باشد، ربات به شما اطلاع می‌دهد که در حال تقسیم‌بندی است، و بعد بخش به بخش پیش می‌رود تا همه چیز را دوباره کنار هم بگذارد.

## نکات و محدودیت‌ها

- یک نشست Colab بی‌نهایت نیست — نسخه رایگان Colab ممکن است بعد از مدتی بی‌فعالیتی یا بعد از حدود ۱۲ ساعت قطع شود؛ بنابراین اگر روی یک حساب رایگان شلوغ، یک فایل خیلی طولانی (مثلاً بیش از ۱۰ ساعت) در حال پردازش باشد، از نظر تئوری ممکن است طول عمر نشست کافی نباشد. استفاده از Colab Pro یا فعال نگه‌داشتن تب مرورگر این ریسک را کم می‌کند.
- خروجی این ربات برای **زبان فارسی** بهینه شده است — ترجمه، زیرنویس و خلاصه‌ها همیشه به فارسی تولید می‌شوند، صرف‌نظر از زبان اصلی فایل صوتی، چون این کاربرد اصلی ربات است.
- فایل‌های session (`*.session`) وضعیت زنده‌ی ورود ربات به تلگرام شما را در خود دارند — هرگز آن‌ها را commit نکنید. این فایل‌ها از قبل در `.gitignore` مستثنا شده‌اند.
